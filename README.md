
## Contexte du Projet
- Un système accessible via la ligne de commande
- Une gestion stricte des permissions selon les rôles
- Une architecture évolutive pour une future interface web
- Un outil de formation pour leurs équipes éditoriales

### Compétences Techniques POO
- **Héritage** : Créer une hiérarchie de classes cohérente
- **Encapsulation** : Protéger les données avec private/protected
- **Polymorphisme** : Utiliser des méthodes communes avec comportements différents
- **Relations objets** : Gérer les associations entre classes
- **Typage PHP 8** : Utiliser les types stricts pour la robustesse


## Fonctionnalités

- **Authentification** avec 4 rôles (Admin, Éditeur, Auteur, Visiteur)
- **Gestion articles** (création, publication, supprimer, modifier)
- **Gestion catégories** (création)
- **Gestion utilisateurs** (ajouter, modifier, supprimer)
